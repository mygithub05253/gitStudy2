깃(Git)
=> 로컬 저장소

깃허브(Github)
=> 원격 저장소로 활용되는 서버

깃허브(원격 저장소)와 로컬 저장소 연결
1. 원격 저장소 만들기
=> 깃허브 로그인 -> repository 만들기

2. 내가 만든 원격 저장소와 로컬 저장소 연결
=> git branch -M main : 브랜치를 main으로 변경
=> git remote 별칭 원격 저장소 주소 : git 로컬 저장소와 연결된 원격 저장소를 설정
: 원격 저장소 주소가 너무 길기 때문에 origin이라는 별칭을 지어줌
: origin 자리에는 원하는 별칭 아무거나 작성해도 상관없음
: 그냥 기본값은 origin이라고 생각해도 상관없음

git remote : 해당 레포지토리와 연결된 원격 저장소를 출력

git push 별칭 브랜치 이름
=> EX) git push origin main : origin이라는 별칭을 가진 원격 저장소의 main이라는 브랜치에다가 업로드

-u 옵션
=> git push -u 별칭 브랜치 이름
=> 해당 옵션을 사용하면 별칭과 브랜치 이름은 생략 가능
