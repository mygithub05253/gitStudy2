커밋 정보 확인하기
=> 소스트리 -> 브랜치
=> cli -> git log
		--graph 옵션 : 그래프 모양으로 커밋 보여주기
		--oneline 옵션 : 커밋 id와 메시지를 한 줄에 출력
		-숫자 : 최근 해당 숫자만큼의 커밋 내용 출력

충돌
=> 같은 파일 내에서 같은 줄을 수정하거나 삭제하거나 변경한 후, 해당 대상을 하나로 합칠 때 충돌이 발생
=> 충돌이 발생되면 자동으로 합쳐진 내용이 커밋되지는 않고 직접 내용을 수정한 후에 똑같이 stage에 올려주고, commit을 해줘야 함

re-base
=> branch를 합치는 두 번째 방법
=> base를 다시 설정
=> base : 브랜치가 뻗어나가기 시작하는 커밋
=> 뿌리를 바꿔주고 싶은 브랜치에서 작업
	=> git rebase 뿌리가 존재하는 브랜치명
=> rebase 완료된 후에는 merge하여 합쳐주는 것도 가능
=> rebase 과정에서 충돌
	=> 충돌 내용을 수정한 후 add하여 stage에 올려주고, git rebase --continue를 사용하여 rebase 진행
